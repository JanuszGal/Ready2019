Exercise 7: App deployment

1. Connect to HYD-SCCM

2. Start Internet Explorer InPrivate mode.

3. Navigate to https://portal.azure.com and Sign in with labadmin@<AzureDomainName>.onmicrosoft.com.

4. On the left navigation bar, click All Services > Intune.

5. Click Client apps > Microsoft Store for Business

6. Click Enable and Save.

7. Click the "Open the business store" link (or navigate to https://businessstore.microsoft.com)

8. Click the "Sign-in" button in the top right.

9. Click "accept" for the Microsoft Store for Business EULA prompt.

10. Click Manage > Settings > Distribute. Select Activate for Microsoft Intune and Microsoft Intune Enrollment.

11. In the "Search the store" search bar, type in "Xbox." Select the first Xbox app.

12. Click "Get the app" on the Xbox app page. Approve the EULA popup.

13. Back in the Intune portal, click "Sync" to sync the app purchase.

14. Click Apps. Verify that Xbox shows up as a Microsoft Store for Business app.

15. Click on the Xbox app and select assignments. Click "Add group." Under "Assignment type" select "Uninstall." Click Included groups and select the Autopilot Users group.


16. Open the following URL: https://github.com/Microsoft/Intune-Win32-App-Packaging-Tool

17. Click the green dropdown labeled "Clone or download" and then click Download ZIP.

18. Open the following URL: https://go.skype.com/windows.desktop.download. This will begin a download for Skype (in .exe format).

19. In the downloads folder, unzip the Intune-Win32 app packaging tool. Create a new folder in the unzipped directory called "Apps." Copy the Skype.exe into this folder.

20. Open the "Intune-Win32-App-Packaging-Tool-master" folder and double click the IntuneWinAppUtil.exe.

21. For the source folder, enter: "C:\Users\Administrator\Downloads\Intune-Win32-App-Packaging-Tool-master\Apps"

22. For the setup file, enter: "C:\Users\Administrator\Downloads\Intune-Win32-App-Packaging-Tool-master\Apps\Skype-8.37.0.98.exe"

23. For the output folder, enter: "C:\Users\Administrator\Downloads\Intune-Win32-App-Packaging-Tool-master\Apps/"

24. Press enter to package the app.

25. Back in the Intune portal, select Apps and +Add.

26. For App type, enter "Windows app (Win32) - preview"

27. For the app package file, select the Skype .intunewin file that was created.

28. Under app information, enter the following information:

a. Description: Skype

b. Publisher: Microsoft

29. For program, enter the following information:

a. Install command: Skype-8.37.0.98.exe /verysilent

b. Uninstall command: Skype-8.37.0.98.exe /uninstall

c. Install behavior: System

30. Under requirements, select 32-bit and 64-bit for Operating System Architecture and Windows 10 1607 for Minimum operating system.

31. Under detection rules, select "Manually configure detection rules." Click Add. Type in the following information for the detection rule:

a. Rule type: File

b. Path: "C:\Program Files (x86)\Microsoft\Skype for Desktop\"

c. File or folder: "C:\Program Files (x86)\Microsoft\Skype for Desktop\Skype.exe"

d. Detection method: File or folder exists

32. Click Ok twice and Add.

33. Once the app is uploaded, select Assignments and assign it to the Autopilot Users group as required.

34. Open the following URL to download an .msi for Power BI Desktop: https://www.microsoft.com/en-us/download/details.aspx?id=45331

35. Select the PBIDesktop.msi option and click next:

36. Back in the Intune portal, select Apps and click +Add.

37. For app type, select “Line-of-business app”. In the app package file section select the PBIDesktop.msi.

38. Under App information, enter the following information:

a. Description: Power BI Desktop

b. Publisher: Microsoft

c. Command Line Arguements: ACCEPT_EULA=1

39. Click Add and then assign the app to the Autopilot users group as required.

Exercise 11: Set up Co-management

1. Connect to HYD-AD

2. Get and install the latest version of Azure AD Connect (1.1.819.0 or higher).

3. Launch Azure AD Connect, and then select Configure.

4. On the Additional tasks page, select Configure device options, and then select Next.

5. On the Overview page, select Next.

6. On the Connect to Azure AD page, enter the credentials of a global administrator for your Azure AD tenant.

7. On the Device options page, select Configure Hybrid Azure AD join, and then select Next.

8. On the SCP page, for each on-premises forest you want Azure AD Connect to configure the service connection point (SCP), do the following steps, and then select Next:

a. Select the forest.

b. Select the authentication service. If you have a federated domain, select AD FS server unless your organization has exclusively Windows 10 clients and you have configured computer/device sync or your organization is using SeamlessSSO.

c. Click Add to enter the enterprise administrator credentials.

9. On the Device operating systems page, select the operating systems used by devices in your Active Directory environment, and then select Next.

10. On the Federation configuration page, enter the credentials of your AD FS administrator, and then select Next.

11. On the Ready to configure page, select Configure.

12. On the Configuration complete page, select Exit.

13. Open the Configuration Manager console > Administration > Overview > Client Settings, and then edit the Default Client Settings.

14. Select Cloud Services.

15. On the Default Settings page, set Automatically register new Windows 10 domain joined devices with Azure Active Directory to = Yes.

16. Select OK to save this configuration.

17. In the Configuration Manager console, go to Administration > Overview > Cloud Services > Co-management.

18. On the Home tab, in the Manage group, select Configure co-management to open the Co-management Configuration Wizard.

19. On the Subscription page, select Sign In and sign in to your Intune tenant, and then select Next.

20. On the Enablement page, from the Automatic enrollment in Intune dropdown list, select one of the following options:

21. Pilot - (Recommended) Members of the collection you specify are automatically enrolled into Intune and can then be co-managed. You specify the pilot collection on the Staging page of this wizard. This option allows you to test co-management on a subset of clients. You can then roll out co-management to additional clients using a phased approach.

22. All - Co-management is enabled for all clients.

23. On the Workloads page you can switch workloads from Configuration Manager to one of the following, and then when ready to continue, select Next.

24. Pilot Intune - Switches a workload only for the devices in the Pilot group. You’ll assign a collection as the Pilot group on the next page of the wizard.

25. Intune - Switches the associated workload for all co-managed Windows 10 devices.

26. You don't need to switch any workloads at the time you enable co-management. You can revisit this configuration from the Configuration Manager console later, after co-management is configured.

27. Before you switch a workload, make sure the corresponding workload in Intune is configured and deployed. Doing so keeps the workloads managed.

28. On the Staging page, specify a collection to use for the Pilot collection, and then click Next. The collection you specify is used as part of your phased rollout of co-management. You can change the collections in the pilot group at any time from the co-management properties.

29. On the Summary page, select Next, and then Close to complete the Wizard.

30. In the Configuration Manager console, go to the Administration workspace, expand Cloud Services, and select the Co-management node.

31. Select the co-management object, and then choose Properties in the ribbon.

32. On the Enablement tab, copy the command line. Paste it into Notepad to save for the next process.

33. Go to the Azure portal, and then open the Intune page.

34. Select Client Apps > Apps > Add.

35. Under Other, select Line-of-business app.

36. Upload the ccmsetup.msi app package file. Find this file in the following folder on the Configuration Manager site server: <ConfigMgr installation directory>\bin\i386.

37. After the app is updated, configure the app information with the command line that you copied from Configuration Manager.

Exercise 15: Exporting settings as a .json file

1. In order to successfully export your settings. Please download the latest Intune PowerShell scripts from here (download and expand the ZIP file):

2. Start an elevated PowerShell session

3. Load the Azure AD Module. (Install-Module AzureAD)

4. Navigate to C:\PSHELL\Intune-scripts\DeviceConfiguration

5. Run the following script "C:\PSHELL\Intune-scripts\DeviceConfiguration\DeviceConfiguration_Export.ps1"

6. Authenticate to the Intune Tenant where you have your configuration

7. All of your Device Configurations will be downloaded

8. Send the contents of the Output folder to a compressed ZIP folder and upload to your Teams site or a OneDrive location for storage.

9. Start an elevated PowerShell session

10. Load the Azure AD Module. (Install-Module AzureAD)

11. Navigate to C:\PSHELL\Intune-scripts\DeviceConfiguration

12. Run the following script "C:\PSHELL\Intune-scripts\DeviceConfiguration\DeviceConfiguration_Import_FromJSON.ps1"

13. Authenticate to the Intune Tenant where you want to import your configuration

14. You will be asked to enter the configuration file for import enter "C:\PSHELL\Output\CustomDevicePolicy.json"

15. Log into or refresh your Intune Device Configuration session and verify you now see the profile. (note the profile will not be assigned.

16. Assign the profile to a well scoped group for production pilot testing.

17. Repeat process for each JSON file you want to import.
